DAO 와 model의 차이

1. DAO(Data Access Object) :

	- 데이터와 어플리케이션 사이의 데이터 접근을 추상적으로 변화시켜 패턴으로 나타낸 class
	- 데이터베이스와 직접 상호작용하는 코드를 분리하고, 데이터베이스 접근을 캡슐화하여 모델과 데이터베이스 간의 결합도를 낮추는 역할을 함
		> 생성, 읽기, 수정, 삭제하기 위한 메서드를 생성 (CRUD를 담는 공간)

2. model :

	- 데이터의 구조와 비즈니스 논리를 표현하는 부분
		> 데이터의 형식, 필드, 속성, 동작, 상호작용 등을 정의
	- 주로 데이터베이스 테이블, 객체, 데이터 구조와 관련됨
